# 更新日志

## [1.1.0] - 2026-05-25

> 从"能用"到"好用"：10 项核心升级，代码量翻倍

### 🧠 智能查重引擎
- **动态哈希策略**：<500MB 文件全量 MD5，>=500MB 三点采样（头中尾各取 1MB），兼顾速度与准确性
- **自动处理**：大小比对 → 哈希比对 → 确认重复则自动删除，不再弹窗打断
- **回收站兜底**：默认调用 send2trash 进回收站，可恢复；勾选"彻底删除"则直接 os.remove（红色警告，默认不勾选）

### 🛡️ 批量下载稳定期检测
- **单文件批量下载**：settle timer 机制——每来新文件重置计时器，3 秒无新文件才批量处理（可配置 3~300 秒）
- **文件夹批量下载**：体积稳定率检测——每隔 2 秒计算文件夹总大小，连续 2 次大小不变且无文件被系统占用，判定下载完成
- 彻底告别脆弱的 .crdownload/.part 临时文件后缀检测，任何下载工具都通用

### 🌐 断网保护机制
- **自动监控**：文件夹准备移动前，socket 连接阿里 DNS (223.5.5.5) 检测网络；断网则挂起并弹窗提示
- **手动整理**：传入 is_manual=True 绕过网络检测，断网状态下也能正常整理本地文件
- 防止下载中途断网被误判为"下载完成"导致文件损坏

### 📂 省时模式升级
- 旧逻辑：跳过无法识别的文件（硬编码 62 个黑名单扩展名）
- 新逻辑：跳过包含子目录的文件夹（如"音乐/周杰伦/"整体跳过，"音乐/林俊杰/"正常处理）
- 文字说明改为"整理时不整理子目录中的文件"

### 🔔 Toast 点击跳转
- 文件移动完成 toast → 点击调用 open_file_location() 定位到目标文件
- 批量文件夹整理 toast → 点击定位到目标文件夹
- 重复文件 toast → 点击定位已有文件位置

### 🎨 托盘 + UI 全面修复
- 托盘左键修复：_TrayIcon 子类化 pystray.Icon，覆盖 _on_notify() 正确拦截 WM_LBUTTONUP
- 托盘右键精简：仅保留"关于"和"退出"
- 窗口居中修复：屏幕尺寸计算 + geometry 定位
- 设置窗口按钮修复：移除最大化按钮，关闭按钮改为 WM_DELETE_WINDOW protocol
- 分类管理 UI：Canvas + Scrollbar 可滚动，每个分类可编辑扩展名/可删除/"其他"受保护
- 标签页顺序：分类管理在第一页，通用设置在第二页

### ⚡ 智能模式归类优化
- 占比 >=70%：不管类型关联性，直接整体归多数派并继承文件夹名
- 占比 <70% 但两类型强关联（如音乐+图片=专辑封面）：整体搬
- 两类型无关联：多数派继承文件夹名，少数派平铺
- 3+ 类型：多数派继承文件夹名，少数派平铺
- 新增 50% 阈值规则：没有任何类型 >=50% 时，全部平铺
- 安装包检测：exe+压缩包同时存在 → 整体归压缩包
